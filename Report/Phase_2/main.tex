\chapter*{\LARGE ABSTRACT}                                       %% ------------- Add the content of your Abstract below ----------------%
\doublespacing
Static Application Security Testing tools remain indispensable in modern software development, yet their persistently high false-positive rates continue to erode developer trust, contributing to widespread alert fatigue and the dangerous habit of ignoring legitimate warnings. Existing efforts to address this problem through machine learning have encountered their own barriers: graph neural networks trained on curated benchmarks fail to generalize when confronted with realistic, deduplicated vulnerability data, and large language models applied to code security produce hallucinated findings and exhibit inconsistency across repeated analyses. These converging limitations point to a fundamental gap --- no single analysis modality is sufficient on its own, and applying expensive techniques uniformly to all findings is both economically wasteful and technically unnecessary.

\indent This project presents \textbf{SEC-C}, a multi-stage cascading framework that treats vulnerability triage as a resource allocation problem. Rather than subjecting every potential finding to the same depth of analysis, the framework applies an uncertainty-driven escalation strategy: a composite scoring model weighing analysis confidence, code complexity, vulnerability novelty, and inter-tool agreement determines whether each finding can be resolved cheaply or must be escalated to progressively deeper reasoning. Findings that the static pre-screener can resolve with high confidence are triaged immediately; ambiguous cases advance to a graph-based structural validator built on Graph Isomorphism Networks and conformal prediction, which provides distribution-free coverage guarantees on its classification sets. Findings that remain unresolved after structural analysis enter a dual-agent large language model stage, where an attacker agent attempts to construct exploit paths while a defender agent identifies sanitizers, access controls, and framework protections. A structured consensus protocol reconciles these adversarial perspectives into a final verdict accompanied by automatically generated CVSS base scores. The cascade concludes with a CWE-adaptive score fusion engine that assigns category-specific weights to each stage's evidence --- recognizing, for instance, that injection vulnerabilities benefit most from semantic understanding while cryptographic weaknesses are best detected by pattern matching.

\indent A distinguishing aspect of the framework is its integration of conformal prediction into the vulnerability detection pipeline, an application that, to our knowledge, has not been previously explored. By calibrating the graph neural network's softmax outputs through a post-hoc temperature scaling procedure, the system produces prediction sets with formal coverage guarantees, enabling principled decisions about which findings warrant expensive language model analysis and which can be resolved with structural evidence alone. The entire cascade is designed for graceful degradation: missing components cause findings to escalate rather than be silently dropped, and the system operates within free-tier API constraints, making the approach accessible for academic and open-source use. Evaluation across multiple open-source repositories and a curated multi-language test suite demonstrates that the majority of findings are resolved at the earliest and cheapest stage, substantially reducing the cost and latency of the overall analysis while maintaining complete coverage of the input finding set.\\

\textbf{Keywords}: vulnerability detection, uncertainty quantification, cascade architecture, conformal prediction, graph neural networks, multi-agent consensus, static analysis, CWE-adaptive fusion
