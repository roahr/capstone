\chapter*{\LARGE ABSTRACT}
%% ------------- Add the content of your Abstract below ----------------%
Static Application Security Testing tools traditionally suffer from high false-positive rates, leading to significant developer alert fatigue and disengagement. Machine learning approaches introduce their own limitations: graph neural networks trained on curated benchmarks fail to generalize to real-world codebases, while large language models hallucinate vulnerabilities and produce inconsistent verdicts. This project introduces the \textbf{Multi-Stage Code Security Framework for Adaptive Vulnerability Triage and Detection}, an uncertainty-driven hierarchical framework designed to address these critical limitations by treating vulnerability triage as a resource allocation problem.

\indent The framework employs a four-stage cascading architecture that progressively escalates only ambiguous findings to more expensive analysis modules. The cascade begins with lightweight syntactic pattern matching and interprocedural taint analysis, proceeds to Code Property Graph construction with Graph Isomorphism Network classification wrapped in conformal prediction sets, advances to a dual-agent large language model consensus protocol employing adversarial attacker and defender roles, and concludes with CWE-adaptive score fusion that produces weighted severity assessments. An uncertainty-driven escalation strategy resolves clear findings at the cheapest stage while routing genuinely ambiguous cases to deeper analysis.

\indent A key innovation is the uncertainty quantification engine, which fuses analysis confidence, code complexity, pattern novelty, and inter-tool conflict factors to compute an adaptive escalation score for each potential finding. The conformal prediction layer provides formal coverage guarantees on graph-based classifications, replacing brittle point estimates with statistically rigorous prediction sets. The adversarial multi-agent validation replaces single-model inference with structured debate, reducing hallucination through independent reasoning chains. Together, these contributions establish a foundation for intelligent, cost-aware vulnerability triage.

\vspace{0.5em}
\noindent\textbf{Keywords:} static application security testing, vulnerability detection, uncertainty quantification, conformal prediction, graph neural networks, large language models, multi-agent consensus, cascaded analysis, code property graphs, CWE-adaptive fusion.
