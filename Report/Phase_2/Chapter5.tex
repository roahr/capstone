\section{\uppercase{Summary of Contributions}}

This work addressed the false-positive problem in static analysis
(68--95\% in practice~\cite{ghostsecurity2025, siftingnoise2026}) through a
multi-stage cascade where each finding is resolved at the cheapest stage
capable of producing a confident verdict. Table~\ref{tab:contributions}
summarizes the five contributions.

Together, these contributions demonstrate that principled escalation, grounded
in uncertainty quantification and conformal prediction (which provides coverage guarantees under the exchangeability assumption), dramatically reduces
analysis cost while maintaining full finding resolution. The uncertainty score
(C1) filters 85\% of findings at the SAST stage, reducing estimated per-scan
cost from \$10--40 (LLM-only) to \$0.50--2.00. The live cascade (C5) confirmed
end-to-end viability: 184 findings across 15 repositories, every one resolved.

\section{\uppercase{Limitations and Future Work}}

We identify six limitations, each paired with a planned mitigation:

\begin{itemize}
    \item \textbf{GNN accuracy vs.\ conformal trade-off.} Removing label smoothing for ConfTS reduced test F1 from 0.781 to 0.750. \textit{Fix:} cross-validated temperature search ($T \in [0.05, 3.0]$, 5-fold) in V6.
    \item \textbf{Language imbalance.} C/C++ constitutes 94.6\% of training data. \textit{Fix:} PrimeVul integration plus targeted collection via Juliet~\cite{juliet2017}, WebGoat, and NodeGoat.
    \item \textbf{Conformal coverage gap.} Offline coverage reached 84.3\%, approaching the 90\% theoretical target and already demonstrating meaningful uncertainty-aware routing. \textit{Fix:} systematic comparison of Platt scaling, histogram binning, and isotonic regression against ConfTS to close the remaining gap.
    \item \textbf{Expert-configured fusion weights.} The 14 CWE weight profiles are configured based on domain expertise rather than empirical optimization. \textit{Fix:} Bayesian optimization over per-CWE simplices using a labeled evaluation set.
    \item \textbf{Python-only end-to-end validation.} The full pipeline was validated primarily on Python targets. \textit{Fix:} per-language integration test suites covering C/C++, JavaScript, Java, and Go.
    \item \textbf{No OWASP Benchmark evaluation.} External validity against the standard 2,740-case SAST benchmark remains outstanding. \textit{Fix:} OWASP Benchmark evaluation to enable direct comparison with commercial tools.
\end{itemize}

Two longer-term directions merit attention: Dynamic Application Security Testing (DAST) integration as an optional fifth cascade stage for runtime exploit confirmation, and IDE extensions (VS Code, JetBrains) that surface SAST-stage annotations in real time~\cite{johnson2013}.

\section{\uppercase{Closing Remarks}}

The central principle of this work, that uncertainty quantification should
drive the routing of findings between analysis modalities, proved effective in
practice. Intelligent escalation between cheap pattern matching, graph-based
structural analysis, and expensive semantic reasoning achieves a cost-accuracy
balance that no single modality can match alone. We release the proposed
framework as open-source software and welcome contributions from the security
research community.
