\chapter{\uppercase{Background and Related Work}}

The preceding chapter established that static analysis tools suffer from
false positive rates exceeding 90\% at scale~\cite{ghostsecurity2025},
and that alert fatigue leaves nearly half of discovered vulnerabilities
unpatched for over twelve months~\cite{edgescan2025}. This chapter surveys
four areas informing the proposed framework: graph neural networks for
vulnerability detection (Section~\ref{sec:gnn-lit}), large language models for
code security (Section~\ref{sec:llm-lit}), conformal prediction
(Section~\ref{sec:conformal-lit}), and hybrid cascaded systems
(Section~\ref{sec:hybrid-lit}). We then identify four research gaps
(Section~\ref{sec:gaps}) that map directly to our contributions.

\section{\uppercase{GNN-Based Vulnerability Detection}}
\label{sec:gnn-lit}

Traditional machine-learning approaches treat source code as a flat
token sequence, discarding structural relationships (control flow, data
dependence, function calls) that determine whether a pattern is
exploitable~\cite{thapa2022}. Graph neural networks operate directly on
Code Property Graphs (CPGs)~\cite{cpg2014}, propagating information
along edges to learn representations encoding both syntactic form and
semantic relationships.

Devign~\cite{devign2019} trained a Gated Graph Neural Network on joint
Abstract Syntax Tree (AST), Control Flow Graph (CFG), Data Flow Graph (DFG), and Natural Code Sequence (NCS) graphs from 27,318 functions, achieving 63.69\%
accuracy but was limited by fixed-iteration message passing.
Reveal~\cite{reveal2021} addressed class imbalance with triplet-loss
training, showing that dataset quality matters as much as architecture.
IVDetect~\cite{ivdetect2021} introduced interpretation-based detection
through program slicing and GNN attention weights, providing line-level
localization cues for developer triage. We adopt a similar principle: our
node importance scores derived from GIN embeddings provide localization information
propagated to the final report.

ReGVD~\cite{regvd2022} showed that Graph Isomorphism Networks with
code-gadget representations achieved competitive F1 scores
($\sim$0.62--0.68 on Devign). Xu et al.~\cite{xu2019gin} proved that
GIN's injective sum aggregation is as expressive as the
Weisfeiler-Leman graph isomorphism test, the theoretical ceiling for
message-passing GNNs. We adopt GIN's injective aggregation with
attention-based readout to retain localization capability.

LineVul~\cite{linevul2022} applied a RoBERTa transformer to line-level
prediction without CPG extraction, but treating code as a linear
sequence discards structural distinctions such as sanitized versus
unsanitized flows. GraphCodeBERT~\cite{graphcodebert2021} bridges this
gap by incorporating data-flow information into pre-training. We use
GraphCodeBERT as our feature extractor: each node receives a 768-dim
embedding concatenated with 6 structural features, yielding
774-dimensional input vectors for our GIN classifier.

PrimeVul~\cite{primevul2024} delivered the most consequential recent
finding: a 7B-parameter model scoring 68.26\% F1 on BigVul achieved
only 3.09\% F1 on rigorously deduplicated data. This motivates our use
of conformal prediction (Section~\ref{sec:conformal-lit}), which provides
coverage guarantees independent of raw accuracy.

\section{\uppercase{Large Language Models for Code Security}}
\label{sec:llm-lit}

LLM-based vulnerability detection follows two paradigms:
\emph{classification} (labeling code as vulnerable or safe) and
\emph{reasoning} (explaining exploitability). The classification paradigm
inherits the benchmark-inflation problem: GPT-3.5 and GPT-4 performed
``akin to random guessing'' on PrimeVul's most stringent
setting~\cite{primevul2024}.

LLM4Vuln~\cite{llm4vuln2024} decomposes vulnerability reasoning into
focused sub-tasks, reducing hallucination over single-prompt approaches.
We adopt a similar decomposition: our Attacker and Defender agents each
focus on a specific analytical perspective.
Steenhoek et al.~\cite{steenhoek2024} confirmed that LLMs exhibit
47--76\% inconsistency across repeated runs, underscoring the need for
consensus mechanisms. VulnHuntr~\cite{vulnhuntr2024} demonstrated that
LLM-driven autonomous hunting can discover real zero-day
vulnerabilities, but its lack of structured validation means findings
still require manual confirmation.

Single-model inference suffers from confirmation bias: once the model
commits to an assessment, subsequent reasoning reinforces rather than
challenges it. Du et al.~\cite{multiagentdebate2023} showed that
multi-agent debate improves factual accuracy by 20--40\%, and
Liang et al.~\cite{divergentthinking2023} extended this with a
``tit for tat'' protocol showing that productive disagreement yields
better outcomes than artificial consensus. These findings motivate our
Attacker vs.\ Defender design, mapping to the red team / blue team
methodology standard in security assessment but, to our knowledge, never
applied to automated vulnerability triage.

We supplement LLM reasoning with retrieval-augmented generation (RAG)
over 200,000+ NVD entries~\cite{nvd2023} and 900+ CWE definitions,
indexed via hybrid FAISS and BM25 retrieval with Reciprocal Rank Fusion.
Both agents receive identical domain-specific context, grounding analysis
in factual knowledge rather than parametric memory.

The central tension is that LLMs excel at explaining \emph{why} code
might be vulnerable but fail at reliably determining \emph{whether} it
is. Our framework exploits this asymmetry by using the LLM as a
reasoning engine for the hardest cases, those surviving SAST and GNN
stages, rather than as a first-line classifier.

\section{\uppercase{Conformal Prediction and Calibrated Uncertainty}}
\label{sec:conformal-lit}

Modern deep neural networks are poorly calibrated.
Guo et al.~\cite{guocalibration2017} showed that as network depth and
width increase, high-confidence predictions are frequently wrong.
Post-hoc methods such as temperature scaling improve calibration on
average but provide no per-instance guarantees.

Conformal prediction, formalized by Vovk et al.~\cite{vovk2005}, offers
a fundamentally different approach: rather than a single prediction, it
outputs a \emph{prediction set} guaranteed to contain the true label
with user-specified probability:
%
\begin{equation}
P\bigl(Y_{\text{true}} \in \mathcal{C}(X)\bigr) \geq 1 - \alpha
\label{eq:conformal_guarantee}
\end{equation}
%
\noindent where $\mathcal{C}(X)$ is the prediction set and $\alpha$ is
the error rate. This guarantee is \emph{distribution-free}, holds in
finite samples, and requires only exchangeability between calibration and
test data~\cite{angelopoulos2023}.

Romano et al.~\cite{romano2020} introduced Adaptive Prediction Sets
(APS), using sorted softmax probabilities as conformity scores and
accumulating classes until cumulative probability exceeds $1 - \hat{q}$.
APS produces smaller sets for confident inputs and larger sets for
uncertain ones. In our cascade, a singleton set resolves the finding at
Stage~2, while a multi-element set triggers escalation to the LLM stage;
the singleton rate directly determines cascade efficiency.

To our knowledge, no published work has applied conformal prediction to
vulnerability detection. Our application of APS to the GNN output layer,
using singleton versus multi-label sets to drive cascade escalation,
represents the first such application in the code security domain.

\section{\uppercase{Hybrid and Cascaded Detection Approaches}}
\label{sec:hybrid-lit}

The limitations in Section~\ref{sec:gnn-lit} and Section~\ref{sec:llm-lit} motivate
combining multiple analysis modalities. No single technique dominates
across all vulnerability types; a multi-stage architecture routing
findings to the appropriate depth can exploit each modality's strengths.

IRIS~\cite{iris2025} applies GPT-4 to all CodeQL findings;
Vulnhalla~\cite{vulnhalla2025} filters CodeQL's haystack with LLM
adjudication; ZeroFalse~\cite{zerofalse2025} sends every SAST trace to
an LLM; LLMxCPG~\cite{llmxcpg2025} augments LLM reasoning with CPG
context. All four process findings uniformly without routing, lack
conformal prediction, and offer no CWE-specific fusion.

\section{\uppercase{Research Gaps and Positioning}}
\label{sec:gaps}

The preceding sections reveal four concrete gaps, each motivating a
corresponding contribution.

\textbf{Gap~1: No Selective Escalation Between Analysis Modalities.}
Every hybrid system reviewed applies its most expensive analysis
uniformly, yet the vast majority of SAST findings are clearly true or
false positives. Our \textbf{Contribution~1} addresses this with a
4-factor uncertainty score ($U = 0.4 \cdot U_{\text{conf}} + 0.3 \cdot
U_{\text{comp}} + 0.2 \cdot U_{\text{nov}} + 0.1 \cdot
U_{\text{confl}} + \delta_{\text{sev}}$) that routes findings through a
3-stage cascade, resolving $\sim$75--80\% at the SAST stage and reducing
LLM calls by approximately 85\%.

\textbf{Gap~2: No Calibrated Uncertainty in Vulnerability Detection.}
All detectors produce point predictions without formal calibration.
Guo et al.~\cite{guocalibration2017} showed neural confidence scores are
systematically miscalibrated; PrimeVul~\cite{primevul2024} demonstrated
that vulnerability models are particularly fragile on out-of-distribution
data. Our \textbf{Contribution~2} applies APS conformal
prediction~\cite{romano2020} to the GNN output layer, providing the
guarantee in Equation~\ref{eq:conformal_guarantee} and replacing
arbitrary thresholds with a principled decision boundary.

\textbf{Gap~3: LLM Triage Lacks Adversarial Validation.}
Existing LLM triage uses a single model or single-perspective prompt,
inheriting confirmation bias~\cite{steenhoek2024}. Multi-agent
debate~\cite{multiagentdebate2023, divergentthinking2023} improves
accuracy by 20--40\% but has not been applied to code security. Our
\textbf{Contribution~3} introduces an adversarial dual-agent protocol:
an Attacker constructs exploits while a Defender identifies mitigations,
and a consensus engine reconciles their perspectives with structured
evidence.

\textbf{Gap~4: Fixed Fusion Weights Ignore CWE-Specific Behavior.}
Hybrid systems use fixed combination strategies, ignoring the empirical
fact that injection vulnerabilities benefit most from LLM reasoning,
cryptographic weaknesses from SAST, and memory safety issues from graph
analysis. Our \textbf{Contribution~4} uses CWE-adaptive score fusion
($f = \alpha \cdot S_{\text{SAST}} + \beta \cdot S_{\text{GIN}} +
\gamma \cdot S_{\text{LLM}}$) with per-CWE configured weights based on domain expertise.

\section{\uppercase{Summary}}
\label{sec:litsurvey-summary}

The four gaps identified in this chapter, selective escalation, calibrated uncertainty, adversarial validation, and CWE-adaptive fusion, remain unaddressed by any single existing system. The next chapter presents our methodology for closing each gap.
